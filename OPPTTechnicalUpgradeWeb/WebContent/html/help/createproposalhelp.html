Create proposal help

Use this form to input the Create Proposal data.

Sales area & currency
Sales organization Organizational unit responsible for the sale of certain products of services, e. g. 0365 Germany.
Distribution channel Manner in which products or services are sold to the customer
Example: A - IBM Sales, F - Fee Based, P - Internals.
Divisions Organizational unit that groups materials, products or services. The division is used to determine the sales area and business area. Example: 00 - IBM Generic
Currency Code Unit of currency. Autoatically applied with sales organization is selected.

Basic proposal data
Proposal name Name of proposal. This is the name under which the proposal is saved and retrieved in OPPT
Proposal type Type of proposal.
Example: ABA - Annual Billing Agreement.
EAS - Enterprise Agreement for S/390 Software.
Report name Name of report. If the user inputs any value in the field, it will be printed as the Report name of the OPPT Report (CAR, Inventory Report etc.)
External contract numer User defined external contract number that will be printed on the invoice. Required Input for all sales organizations except US.
Proposal content Select one.

Contract dates
Contract start date Date contract starts.
When you select the contract start date, the system will default the contract end date, contract period beginning, and contract period ending based on the contract start date. The dates can be modified.
Contract end date Date contract ends.
Contract period begining Start date of the contract billing period
Contract period ending End date of the contract billing period.

Pricing information
Planned growth Uplift for EAS, EGO (except US sales org 0147), NUEV and /or SGA type of contracts. Will be reflected in the calculated price. If NIL occurs no data can be entered in this field.
Billing frequency Frequency of billing.
Example:
M – monthly
Q – quarterly
S – semi-annually
A – annually.
Growth limit Growth Allowance for EAS, EGO (except US sales org 0147), NUEV and /or SGA type of contracts. Will be reflected in the calculated price. If NIL occurs no data can be entered in this field.
Advance Billing to be in advance (ticked) or in arrears (not ticked). Default is ticked.
PAKID If invoice is to be considered for products sourced from different pipes plus composite contracts only (e. g. where there are non-S/390components processed outside OPPT/ESW). For UK/Ireland and Netherlands only.
TVM rate Time value of money. Allowed for SGA if billing frequency is annual.
MLC Anniversary contract amount Monthly lease contract: TP, ESSO, ELITE, EGO, EAS, AMO and NUEV contracts only. Actual amount the customer will be invoiced for the year. Revenue is recognized by a proportional split of this bottom-line price over the base inventory of the contract.
The anniversary amount entered should be divisible by billing frequency, i.e. for monthly = 12, quarterly = 4, semi-annual = 2. No decimals allowed.
S&S Anniversary contract amount Service & Subscription: TP, ESSO, ELITE, EGO, EAS, AMO and NUEV contracts only. Actual amount the customer will be invoiced for the year. Revenue is recognized by a proportional split of this bottom-line price over the base inventory of the contract.
The anniversary amount entered should be divisible by billing frequency, i.e. for monthly = 12, quarterly = 4, semi-annual = 2.
Detailed invoice Indicator for summary and detail invoicing for contract billing. If ticked, detailed invoice will be generated (US only).
GSA Indicator will identify contracts that fall under GSA Schedule for US Federal customers (Customer Class 12 and 15).
Only valid for Sales Organization 0147 (US)
Billing block Enter a billing block if contract needs to be blocked for billing, e.g. 07- Pending Addtl. Info, 03 Pricing incomplete etc...
Note: When contract is ready for billing the billing block needs to be removed in ESW.
Deviating date rule The default value is blank. The user can select this field when he/she wants to determine a different billing date for special customers/situations.
Example: French Government customers has to be billed on the 5 th of the month.
The selection criteria can be equal to the values of the Billing Date Rule field.
Billing date rule Date Determination rules for the contract billing plan. If nothing is selected, the ESW default will apply.
Examples: YA = ESW annual advance, YQ = IBM ESW quarterly advance. All other can be checked out on the field’s drop down menu.
Free text for invoice print User is able to input a text up to 6000 characters that will be printed on the invoice. Available for all GEOs except US (Sales Org. 0147).

Purchase order data
P.O. number This field is for input of the customer’s purchase order number (customer authorization). Content of this field will be printed on the invoice.
P.O. date Date of the purchase order.
Content of this field will be printed on the invoice.

Payment terms
Add value days The number of days between the billing date and the date on which the terms of payment for the sales document become effective.
Fixed val. date Fixed date from which payment term period commences.
Payment terms Payment terms for the contract. If nothing is selected, the ESW default from MPP will apply.
Example: A001 (Zero days for the due date/LPF date calculation) etc.

Variance
CAP amount ESSO and ELITE contracts only, amount that contract value can fluctuate prior to amendment.
- % Percentage a contract can fluctuate below its initial value prior to amendment.
+ % Percentage a contract can fluctuate above its initial value prior to amendment.

Financing data
CAL # Used only when the contract is being financed by CAL #

Comments
Comments Remarks in field ‘Comments’ are for user only. Content of this field will not transmit anywhere. It's for SPC or Contract Admin own use (so long as proposal status).

Action buttons
Press Continue button after you finished data entry.
Press Clear all button will discard all you input.
Press Cancel button to go back to proposal maintenance page.

Close window
